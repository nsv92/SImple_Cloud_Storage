Сетевая часть проекта - Netty.

И клиент, и сервер - консольные приложения.

Для обмена информацией исспользуются специальные классы-обертки (в пакете Messages), проходящие через ObjectDecoder/Encoder. Это накладывает ограничение на размер пересылаемого файла. В дальнейшем попытаюсь уйти от этого, с помощью интерфейса ChunkedInput.

База данных пользователей - MySQL. Hеализованна минимальная необходимая функциональность: регистрация пользователя, логин уже зарегистрированных пользователей.

Работа с файлами и фс. Релизовано создание индивидуальной директории для каждого пользователя на серверной и на клиентской стороне, а так же отправка файла с клиента на сервер с помощью комнды /upload, скачивание файла с сервера на клиент (/download),
просмотр файлов и папок на серверной стороне (/dircontent).
